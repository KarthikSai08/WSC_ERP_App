🔑 Idempotency — Duplicate Request Prevention
High Priority

Prevent clients from accidentally creating duplicate resources on retry. The client sends a unique Idempotency-Key header. The server caches the first response in Redis; subsequent identical requests return the cached response without re-executing the handler.

  • Create IdempotencyMiddleware.cs — extract Idempotency-Key header
  • On first request: execute + store result in Redis (keyed by Idempotency-Key)
  • On repeat request: return cached response (HTTP 200 with original body)
  • Set Redis TTL (e.g. 24 h) for idempotency records
  • Apply only to mutating endpoints (POST / PUT / PATCH)
